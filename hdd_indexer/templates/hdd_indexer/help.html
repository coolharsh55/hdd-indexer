Quick Start

    {% if welcome == True %}
    Welcome to HDD-indexer.
    {% endif %}
  • 1 Start the server with python hdd-indexer.py
  • 2 Look for the opened localhost:8000 in your browser
  • 3 Configure settings if required to make sure your HDD Root is configured properly according to the attached HDD.
    On *NIX, it will be something like /Volumes/HDD;
    On Windows, it will be a drive letter like G:.
  • 4 The Movie Folder path is the directory containing your movies. Make sure the path is correct, and that it is a subdirectory of the HDD Root.
  • 5 Start crawler, and wait while it identifies movies stored in your movie folder.
  • 6 Start loader, and wait while it fetches metadata from online sources. It uses multi-threading, so you may experience a slight performance loss.
  • 7 You can now browse movies and its related metadata at localhost:8000/admin/movies/.
    Use the username and password given to you during the setup. Once logged in, you can click on movies to access its metadata.

About HDD-indexer

HDD-indexer is an utility that indexes a hard disk's contents, and allows the user to view it using organisation based on its metadata.
Currently, it supports only movie files, and downloading related metadata from online sources.

Browse Movies

The browse module allows viewing of identified movies based on organisation based on their metadata. It uses a combination of dynamic webpages and django's admin interface to allow you to view stored metadata.
If using the admin interface, use the username and password generated for you during the setup.
Browsing the movies is simple. You are shown a list of movies by default. You can filter it using various paremeters such as release date, director, and actor. Sorting is also supported based on date, cast and crew, and ratings.
If you need to edit a particular movie, you can do so by clicking on it, and changing the information on the form generated for you.
Please be careful when changing data, since the system assumes you know what you are doing, and its effects.

Comparator

not implemented yet
The contents of two remote or local disks can be compared with the Comparator module.
It can match movies on different disks, and create a list of movies that are missing on the current disk which can be borrowed from the other disk.
Alternatively, it can create a lend list for movies present on the current disk, which is absent on the other disk.

Crawler

The crawler looks for movies on the hard disk based on file extensions and file size.
Identified movies are saved to the database along with their relative path from the movie folder.
Running the crawler can take time if there are a large number of files and folders to assess.
Movies identified through the crawler have their title set to the filename minus the extenion, and their relative path set to their file path.

Database

HDD-indexer uses a locally stored SQLite database, named db.sqlite3 situated in the hdd_indexer folder.
This database can be accessed and modified by any other process or program if required.
The database contains all the program data and settings. If you wish to backup these, you can simply copy the database to another location.

Exporter

not implemented yet
The Exporter module can be used to export a list of movies in database and on disk.
The exported list can be saved in many formats, such as csv, txt, json, xml, pdf, binary, etc.

Loader

The Loader module is used for downloading metadata from online source for movies in database.
It uses TMDb v3, OMDb, and OpenSubtitles to retrieve movie information.
TMDb, and OMDb are online movie databases that provide metadata such as title, release date, cast, crew, ratings, etc.
OpenSubtitles is used to retrieve the IMDb ID if it cannot be identified by other services. The other metadata is downloaded from other services based on the identified IMDb ID.
can take substantial amount of time
The loader can take a substantial amount of time for downloading metadata. This is normal since requests to external services take time, and are based on the speeds of your internet connection and DNS.

manage.py

The manage.py is a django command (file/module) that allows you to run the server, interact with the database, open a shell for interactions, and lots more.
To use this, type python manage.py followed by the subcommand/help
Some useful options:
  • runserver: start the server at localhost:8000 (can be changed through options)
  • migrate: create and make changes to the database based on changes in the models code
  • shell: start a local shell which has all project settings loaded
  • createsuperuser: create an admin user

Organizer

not implemented yet
The Organizer module can move and arrange the file on disk based on their metadata.
For example, the movies can be automatically put into folders based on genre, directors, or year.

Server

The server is run as a django program at default settings of 192.168.0.1:8000.
In case of port clash, please specify another port when starting the server with python manage.py runserver
Sometimes, the server process does not exit as expected, and exists as a background or a ghost process. In such cases, find and terminate this process before starting it again.
The server will accept requests from other programs or applications. This can be useful for modifications, but is also potentially hazardous based on use.

Settings

The Settings section allows you to change settings such as HDD Name, Root Path, and Movie Folder path.
The settings changed are first validated before being updated in the database.
